# Contributing

Thank you for considering contributing to JD.SemanticKernel.Extensions!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/<you>/JD.SemanticKernel.Extensions.git`
3. Create a branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Run tests: `dotnet test`
6. Push and create a Pull Request

## Development Prerequisites

- [.NET 8.0 SDK](https://dotnet.microsoft.com/download) or later
- [.NET 10.0 SDK](https://dotnet.microsoft.com/download) (for latest TFM)

## Coding Standards

- Follow the `.editorconfig` rules
- All public APIs must have XML documentation
- All changes must include tests
- Use `dotnet format` before committing

## Pull Request Process

1. Ensure all tests pass
2. Update documentation if needed
3. Use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages
4. One feature/fix per PR
